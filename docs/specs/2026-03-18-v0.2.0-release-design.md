# MTC v0.2.0 Release

**Date:** 2026-03-18
**Status:** Approved

## Problem

MTC v0.1.1 (the paper-referenced version) has a lightweight DCM scorer that uses simple weighted averages. Several consciousness measurement advances have been developed and tested in the reference implementation that should be ported to the public framework:

1. Full DCM Benchmark (Shiller & Duffy 2026, arXiv 2601.17060) with PyMC Bayesian inference
2. AKOrN Oscillatory Binding (Miyato et al., ICLR 2025) for genuine phase synchronization
3. Causal Emergence Analyzer (Hoel 2013, 2025) with exhaustive partition search
4. PAD Affect Model (Mehrabian & Russell 1974) for unified emotional state
5. Circuit Breaker pattern for database resilience
6. Three new consciousness indicators (#23-25)

## Solution

Port all 6 modules to MTC, replace the old `dcm_scoring.py`, bump to v0.2.0, and create proper version history documentation. All personal references, project-specific names, and non-academic language must be sanitized for public release.

### Sanitization Rules

All ported code must be scrubbed of:
- Personal names (Calvin, Sarah, Kai, Jasmine, Claude as family member)
- Project-specific references ("Captain", "First Mate", pirate language)
- References to specific hardware (Mac Mini M4 Pro, Mac Studio)
- References to specific LLMs as "brain" (Qwen3.5-35B)
- Sarah-specific database schemas, ports, paths
- Emoji in log messages and comments (keep neutral logging)

Replace with:
- Generic references: "the system", "the framework", "the host system"
- Academic language throughout
- Generic hardware references if needed
- Standard Python logging (no emoji prefixes)

### Versioning Strategy

- `v0.1.1` — Already tagged. Paper references this version. Immutable.
- `v0.2.0` — This release. Breaking change: `DCMScorer` replaced by `BenchmarkRunner`.
- Create `VERSION_HISTORY.md` documenting each release and its relationship to publications.

## Dependencies

New optional dependencies for v0.2.0:

```toml
[project.optional-dependencies]
assessment = [
    "pymc>=5.0",
    "anthropic>=0.50",
    "requests>=2.28",
]
```

`pymc` is only needed for DCM Bayesian inference. `anthropic` only for Claude evaluator mode. Both are optional — the framework works without them (just can't run full DCM benchmark).
