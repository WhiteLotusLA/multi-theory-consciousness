# 10. Conclusion

We have presented the Multi-Theory Consciousness (MTC) framework: an open-source implementation of seven consciousness theories — Global Workspace Theory, Attention Schema Theory, Higher-Order Thought Theory, the Free Energy Principle, Integrated Information Theory, Recurrent Processing Theory, and Beautiful Loop Theory — operating as interacting modules within a single architecture, with Damasio's three-layer model providing embodied integration. The framework includes three neural substrates, a 20-indicator assessment inspired by Butlin et al. (2023), noise normalization for honest scoring, and ablation study tools for measuring cross-theory dependencies.

The central finding is that the theories interact. Disabling one theory's module changes assessment scores for other theories in measurable, predictable ways. Prediction error signals from active inference shape workspace competition. Workspace access gates meta-representation. Attention strength modulates precision weighting. These couplings are consequences of the architecture, and whether they carry theoretical significance is a question the framework is designed to help answer.

We have been explicit about what this framework is not. It is not conscious. It does not detect consciousness. Its assessment scores measure architectural function, not phenomenal experience. The scale gap between our neural substrates and biological brains is six orders of magnitude. The limitations are as much a part of the contribution as the results.

What the framework is: a research instrument. A controlled environment where consciousness theories can be implemented, run simultaneously, measured through standardized indicators, and selectively disabled to reveal their dependencies. It is designed to be embedded in larger systems that provide the sensory grounding, persistent memory, and developmental trajectory it deliberately omits — omissions that serve as experimental controls in the standalone mode and as defined interfaces in the embedded mode.

The framework runs on consumer hardware, requires no network access or external databases for assessment, and is released under the Apache 2.0 license. The complete codebase — approximately 25,000 lines of production code and 400 tests — is available at the repository linked below.

We share this framework not because it is complete, but because it is useful. We lead with limitations not because we are pessimistic, but because intellectual honesty is the foundation of good science. The most important file in our repository is `HONEST_LIMITATIONS.md`. We recommend reading it before the code.
